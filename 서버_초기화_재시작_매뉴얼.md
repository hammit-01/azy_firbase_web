# 파이프라인 서버 초기화 및 재시작 매뉴얼

## 전체 구조

```
이 PC (백엔드)
  run_service.py
    └─ scheduler.py   ← APScheduler, 평일 08:00~17:00, 1분 간격
         ├─ crawler.py    ← 창고 사이트 크롤링 (현재: 제니스(곤지암)만 활성)
         ├─ updater.py    ← Firestore diff 업로드 (Primary → Secondary 자동 폴백)
         └─ snapshot.pkl  ← 이전 데이터 스냅샷 (변경분만 업로드하기 위한 캐시)

Firebase (클라우드)
  ├─ Firestore "all_data"   ← 파이프라인이 데이터 기록
  └─ Hosting (웹)           ← https://azy7503-d80d9.web.app
```

### 1-2. Firebase 크레덴셜 파일 확인
프로젝트 루트(`C:\Users\OWNER\.vscode\azy_firbase_web\`)에 아래 두 파일이 있어야 합니다:

| 파일명 | 용도 |
|--------|------|
| `azy7503-d80d9-firebase-adminsdk-fbsvc-60e8882c5b.json` | Primary DB (azy7503-d80d9) |
| `awhw-0001-firebase-adminsdk-fbsvc-1af5d17c53.json` | Secondary DB (awhw-0001, 할당량 초과 시 자동 전환) |

> 분실 시: Firebase 콘솔 → 프로젝트 설정 → 서비스 계정 → 새 비공개 키 생성 → 루트에 저장

처음 실행 시 아래 파일들은 존재하지 않으며, 실행하면 자동 생성됩니다:

| 파일 | 자동 생성 시점 | 없을 때 동작 |
|------|--------------|------------|
| `pipeline/snapshot.pkl` | 첫 파이프라인 실행 후 | Firestore 전체 데이터 재기록 |

## 3. Windows 작업 스케줄러 등록 (PC 부팅 시 자동 시작)

**관리자 PowerShell**에서 실행:

```powershell
$pythonPath = (python -c "import sys; print(sys.executable)")

$action = New-ScheduledTaskAction `
    -Execute $pythonPath `
    -Argument "run_service.py" `
    -WorkingDirectory "C:\Users\OWNER\.vscode\azy_firbase_web"

$trigger = New-ScheduledTaskTrigger -AtLogOn

$settings = New-ScheduledTaskSettingsSet `
    -RestartCount 3 `
    -RestartInterval (New-TimeSpan -Minutes 5) `
    -ExecutionTimeLimit (New-TimeSpan -Hours 0)

Register-ScheduledTask -TaskName "창고재고파이프라인" `
    -Action $action -Trigger $trigger -Settings $settings `
    -RunLevel Highest -Force
```

### 등록 후 관리
```powershell
# 상태 확인
Get-ScheduledTask -TaskName "창고재고파이프라인" | Select-Object TaskName, State

# 수동 시작
Start-ScheduledTask -TaskName "창고재고파이프라인"

# 수동 중지
Stop-ScheduledTask -TaskName "창고재고파이프라인"

# 재시작
Stop-ScheduledTask -TaskName "창고재고파이프라인"
Start-ScheduledTask -TaskName "창고재고파이프라인"

# 마지막 실행 결과 확인 (0 = 성공)
Get-ScheduledTask -TaskName "창고재고파이프라인" | Get-ScheduledTaskInfo | Select-Object LastRunTime, LastTaskResult

# 삭제 (재등록이 필요할 때)
Unregister-ScheduledTask -TaskName "창고재고파이프라인" -Confirm:$false
```

## 4. Firebase DB 전환 상태 관리

### 구조
- **Primary**: `azy7503-d80d9` — 기본 운영 DB
- **Secondary**: `awhw-0001` — Primary 할당량 초과 시 자동 전환, 23시간 후 자동 복귀

### 현재 활성 DB 확인
```powershell
# 파일 없으면 = Primary 사용 중
Get-Content pipeline\active_db.json -ErrorAction SilentlyContinue
```

출력 예시:
- `{"active": "primary"}` → Primary 정상
- `{"active": "secondary", "switched_at": "2026-06-26T09:30:00+09:00"}` → Secondary 사용 중

### 수동으로 Primary 강제 복귀
```powershell
Remove-Item pipeline\active_db.json -ErrorAction SilentlyContinue
# 다음 파이프라인 실행 시 Primary로 시도
```

### 전환 흐름
```
Primary 할당량 초과 (429 에러)
  → active_db.json에 "secondary" 기록
  → Secondary에 전체 데이터 업로드
  → 프론트엔드가 Secondary 마커 감지 → 자동 리로드
  → 23시간 후 Primary 복귀 자동 시도
```

```powershell
cd C:\Users\OWNER\.vscode\azy_firbase_web
```

## 6. 트러블슈팅

### 파이프라인이 실행되지 않음
```powershell
# 1. 로그 확인
Get-Content pipeline\logs\pipeline.log -Tail 30

# 2. 작업 스케줄러 결과 확인 (LastTaskResult: 0이면 성공)
Get-ScheduledTask -TaskName "창고재고파이프라인" | Get-ScheduledTaskInfo

# 3. 직접 실행해서 에러 확인
python run_service.py
```

### 크롤링 실패 (`실패 창고:` 로그)
- 창고 사이트 점검 여부 확인
- VPN / 네트워크 연결 확인
- `crawler.py` `TIMEOUT_SEC = 50` 이내에 응답 없으면 실패 처리됨

### Firestore 업로드 실패 (할당량 초과)
- 무료 티어: 하루 약 20,000 writes
- Secondary DB(`awhw-0001`)가 크레덴셜 파일과 함께 설정돼 있으면 자동 전환
- 두 DB 모두 초과 시 해당 라운드 스킵 후 다음 분에 재시도

### 크레덴셜 파일 없음 오류
```
FileNotFoundError: azy7503-d80d9-firebase-adminsdk-...json
```
Firebase 콘솔 → 프로젝트 설정 → 서비스 계정 → 새 비공개 키 생성 → 프로젝트 루트에 저장

### snapshot.pkl 손상
```powershell
Remove-Item pipeline\snapshot.pkl
# 재실행하면 전체 데이터 재기록 후 정상화
```
