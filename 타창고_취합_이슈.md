## 이슈 1. ESTNO 완전 누락 창고

WMS `EST NO` 컬럼이 항상 비어있고 ESTNO가 수탁품 문자열에 내장돼 있음.  
EDA에서 수탁품 파싱으로 추출하는데, 파싱 정규식이 실패하면 ESTNO null.

| 창고 | null 비율 | 수탁품 형태 예시 |
|---|---|---|
| 삼진1 | 13/13 (100%) | `(우)본갈비S/GF(KILCOY)640` |
| 삼진2 | 22/22 (100%) | `(우)등갈비(EXCEL)SE#86M` |
| 신우냉장 | 22/22 (100%) | `우삼겹양지 EXCEL 3990 86E` |
| 삼일물류 | 6/6 (100%) | `GF상우둔CAP OFF(ACC)[000011]` |
| CS | 11/11 (100%) | `(우)안창(GREENLEA)[ME124]` |
| 시에이치물류 | 10/10 (100%) | `A(우)갈비XT/278(1045622)-IBP` |
| 오로라CS | 6/6 (100%) | `(우)업진살UN-SWIFT` |

## 이슈 2. 규격단위중량 오염

WMS `규격` 컬럼에 중량이 아닌 다른 값이 섞여있어 `평균중량` 추출 실패.

| 창고 | 실제 들어오는 값 | 문제 |
|---|---|---|
| 시에이치물류 | `전)5/4용범57개중`, `전산이체`, `전)12/1화남180개` | 재고 메모 텍스트가 규격 컬럼에 들어옴 → 숫자 추출 불가 |
| 오로라CS | `969G/오렌지팜스36.49KG` | ESTNO + 브랜드 + 중량이 한 칸에 합쳐짐 |
| 대재 | `(800)28.10KG28.1KG` | 같은 중량이 두 번 반복 |
| 신우냉장 | `(2506)8.80KG8.8KG` | 박스 수량 + 중량이 합쳐짐 |
| 희창냉장 | `EXCEL28.26KG` | 브랜드 + 중량이 합쳐짐 (eda에서 숫자만 추출하므로 중량은 OK, 브랜드 누락) |

## 이슈 3. 유통기한제조일자 포맷 오염

| 창고 | 원본 WMS 값 | 문제 |
|---|---|---|
| 대재 | `2027.10.292025.10.30` | 유통기한 + 제조일자가 공백 없이 붙어서 옴 |
| 이스트밸리 | `2027.08.182025.08.19` | 동일 |
| 삼일물류 / 신우냉장 | `2028.03.11.  .` | 제조일자 자리에 `. .` (빈 값) |

`eda_common`의 regex `r"(\d{4}\.\d{2}\.\d{2})"` 이 첫 날짜만 추출하므로 유통기한은 맞게 들어가지만,  
제조일자가 붙어있는 경우 `2025.10.30`이 제조일자로 들어갈 수도 있음. 확인 필요.

## 이슈 4. 한라동탄 규격 all null

원본 WMS `규격` 컬럼이 비어있는 행이 대부분 → `규격단위중량` 100% null → `평균중량` 추출 불가.

```
원본: 소비기한=2028.04.16 | 수탁품=(계)장각/SIF530/SEARA | EST NO=SIF530 | 수량=63 | 규격=(없음)
```

## 이슈 5. 행 수 절반 감소 (중복 계정 크롤링)

여러 계정으로 크롤링 → 동일 데이터 중복 → EDA에서 pk 기준 합산.  
창고당 정확히 2배 또는 3배가 나오면 정상. 하지만 **다른 BL이 같은 pk로 합산되는 경우는 실수량 오류 가능.**

| 창고 | 정규화 행 | EDA 행 | 비율 |
|---|---|---|---|
| 대재 | 48 | 16 | 3x |
| 한라곤지암 | 43 | 21 | ~2x |
| 강동2 | 56 | 28 | 2x |
| 신우냉장 | 22 | 11 | 2x |
| SWC | 18 | 9 | 2x |
| 삼일물류 | 6 | 3 | 2x |
| 오로라CS | 6 | 3 | 2x |

## 이슈 6. 대청 빈 응답 행

특정 계정 로그인 시 WMS가 `"조회된 결과가 없습니다."` 1행을 반환.  
현재 `shape[1] <= 2` 체크로 걸러지지만 향후 WMS HTML 구조 바뀌면 해당 텍스트가 데이터로 섞일 수 있음.

## 이슈 7. 경인 ESTNO / 규격 일부 null

| 항목 | null 비율 |
|---|---|
| ESTNO | 3/3 (100%) — WMS EST NO 컬럼 값 비어있음 |
| 규격단위중량 | 2/3 (66%) — 규격 컬럼 값 비어있는 행 있음 |

```
원본: 수탁품=돈목전지(SMITHFIELD)18079 | EST NO=(없음) | 규격=(없음)
```

ESTNO는 수탁품에 내장(`18079`). 규격(중량)이 없는 건 WMS 자체 데이터 미입력.
